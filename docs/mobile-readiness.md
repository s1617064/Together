# 手机正式使用检查清单

这份清单是围绕你们“主要用手机记账”来写的。

## 当前已经具备

### 手机界面

- 底部导航是手机优先布局
- 类目选择是点选式，适合拇指操作
- 首页时间轴是竖向浏览模式
- 记一笔页输入流程短，适合快速记账

### 安装体验

- 已加 `manifest.webmanifest`
- 已加 App 图标资源
- 已加 `theme-color`
- 已加 `apple-mobile-web-app-*` 基础 meta

这意味着：

- Android / Chromium 浏览器更容易“添加到主屏幕”
- iPhone 也能作为网页快捷方式添加到桌面

### 基础离线能力

- 已加 `service-worker.js`
- 会缓存应用外壳文件

这意味着：

- 打开过一次后，再次进入会更稳
- 弱网环境下基础页面更容易打开

### 手机安全区

- 已适配 `safe-area-inset-top`
- 已适配 `safe-area-inset-bottom`

这意味着：

- 刘海屏和底部 Home 指示条区域更不容易挡住内容

## 现在还不算完整正式版的地方

### iPhone 桌面图标

当前图标先用了 SVG 资源，适合继续演示和开发。

如果你们要长期正式使用，建议后面补：

- 180x180 PNG 的 `apple-touch-icon`

### 推送和提醒

现在还没有：

- 每日提醒记账
- 到月提醒
- 预算超额提醒

### 完整离线同步

当前 `service worker` 只是基础壳缓存，不等于完整离线协作。

真正两个人长期使用，更关键的是：

- Firebase 登录
- Firestore 实时同步
- 离线后恢复同步

### 部署方式

现在仍然是本地原型目录打开。

正式给手机长期使用，建议部署到：

- Firebase Hosting

## 最适合的下一步

如果你们真的要开始在手机上长期记账，建议顺序是：

1. 填好 Firebase 配置
2. 把原型部署到 Firebase Hosting
3. 用两部手机分别登录验证同步
4. 再补桌面图标 PNG 和安装提示
