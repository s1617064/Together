# 共同记账 APP MVP 方案

## 目标

做一个只面向两个人使用的共同记账 APP，核心目标是：

- 你和你老婆共同记账
- 每笔账都明确显示是谁记的
- 首页用时间轴展示最近记账动态
- 其他页面保持常规记账逻辑
- 支持网络同步
- 尽量把成本压到最低

## 我建议的版本路线

### 第一阶段：先做 PWA

适合你现在的原因：

- 用户只有两个人，不需要一开始就上复杂架构
- 可以同时覆盖 iPhone 和 Android
- 分享链接就能安装使用
- 开发成本、维护成本、试错成本都最低

### 第二阶段：数据真正联网

推荐组合：

- `Firebase Authentication`
- `Cloud Firestore`
- `Firebase Hosting`

为什么我更推荐它做第一版：

- 一套服务就能把登录、同步、托管做完
- Firestore 很适合共享账本这种“记录流 + 实时同步”模型
- 前期免费额度对两个人使用基本够用
- 后面如果要做消息提醒、图片小票、统计分析，也有扩展空间

## 首页怎么设计最合适

首页建议不是传统报表首页，而是“生活流水时间轴”。

一个比较合适的首页结构：

1. 顶部概览
   显示本月总支出、你记了多少、老婆记了多少
2. 时间轴主体
   按日期倒序展示每一笔记录
3. 每条时间轴卡片
   显示金额、分类、备注、时间、付款人、记账人
4. 快速记账入口
   首页底部悬浮按钮，一点就能记一笔

这样做的好处是：

- 很直观地看到“最近发生了什么”
- 非常适合两个人协同使用
- “谁记的”这件事会天然展示出来，不会藏得太深

## 最低成本方案

### 推荐结论

如果你现在最在意的是“先做出来、先能用、先别花钱”，我建议：

- 先做 `PWA + Firebase 免费层`
- 不急着上原生双端
- 不急着买服务器
- 不急着做管理员后台

### 为什么这条路最省

- 不需要自购云服务器
- 不需要后端常驻运维
- 不需要原生双端同时开发
- 不需要先花钱上应用商店

### 成本大概会落在哪里

前期大概率只有这些可能费用：

- 域名：可选，不买也能先用
- iOS 上架费用：如果以后要进 App Store，再考虑 Apple Developer
- 短信登录费用：如果不用手机号登录，可以先避免

### 官方价格信息

我查了官方资料，当前更适合你这个场景的免费信息是：

- Firebase `Spark plan` 是 no-cost，且不需要 payment method。
- Firebase Authentication 的其他认证服务在免费层可用。
- Cloud Firestore 免费层包含 `1 GiB` 存储、`20K writes/day`、`50K reads/day`。
- Firebase Hosting 免费层包含 `10 GB` 存储和 `360 MB/day` 传输。

对两个人共同记账来说，这个量通常已经很宽松。

## 实时同步为什么能成立

Firestore 适合这个场景，主要因为两点：

- 官方文档说明可以通过 `onSnapshot()` 监听数据变化，内容变更后会继续推送新快照。
- 官方文档说明支持离线缓存，离线时也能写、读、监听和查询，恢复联网后会同步本地变更。

这意味着：

- 你记一笔，老婆那边可以很快看到
- 偶尔没网也能先记，等网络恢复再同步

## 登录方式怎么选最省事

推荐优先级：

1. 邮箱验证码登录
2. Google / Apple 登录
3. 手机号登录

原因：

- 手机号登录常常会产生短信成本
- 两个人使用，邮箱或第三方登录已经足够
- 实现上也更轻

## 数据结构建议

最小可用模型建议这样设计：

### `books`

一本共享账本。

字段示例：

- `name`
- `ownerUid`
- `currency`
- `createdAt`

### `books/{bookId}/members`

账本成员，固定两个人。

字段示例：

- `uid`
- `displayName`
- `role`
- `avatarColor`
- `joinedAt`

### `books/{bookId}/expenses`

每一笔记账记录。

字段示例：

- `amount`
- `category`
- `note`
- `spentAt`
- `payerUid`
- `recordedByUid`
- `recordedByName`
- `createdAt`
- `updatedAt`

关键点：

- `payerUid` 表示这笔钱是谁花的
- `recordedByUid` 表示这笔账是谁记的

这样就能解决你最在意的“软件能显示是谁记的账”。

## 功能优先级

### 第一版必须有

- 两人加入同一本账本
- 新增支出
- 修改 / 删除支出
- 首页时间轴
- 显示记账人
- 实时同步
- 月度汇总

### 第二版再做

- 分类统计图
- 预算提醒
- 小票拍照
- 周报 / 月报
- 搜索和筛选
- 导出 Excel

## 一个很实际的产品判断

如果这个 APP 真的只是你们夫妻两个人长期使用，那我不建议一开始做得太重。

最划算的思路是：

- 先把“记得快、看得清、能同步”做好
- 先把“谁记的、什么时候记的、花在哪儿”做扎实
- 等你们连续用了 1 到 2 个月，再决定要不要继续加预算、报表、图片、小程序、原生客户端
